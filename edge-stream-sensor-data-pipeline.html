Paket Dosya Adı Formatı

packet_{index}_{start_ts_ns}_{end_ts_ns}_{created_at}.parquet

Deterministik packet_id (UUID5)

raw = f"{index}:{start_ts}:{end_ts}:{checksum}"
packet_id = uuid5(NAMESPACE, raw)

Aynı veri → daima aynı UUID. Idempotency garantisi.

Efektif Hz Ölçümü

effective_hz = n_samples / (duration_ns / 1e9)

Her paket sonunda loglanır.

Tasarım

Mimari Kararlar

📦

Neden Parquet + Snappy?

Format Boyut / Paket
CSV (float64) ~1.5 MB
Parquet + Snappy (float32) ~250–300 KB
Azaltma ~%80

Snappy: gzip'e göre çok daha hızlı açılır, edge CPU'yu yormaz.

🗄️

Neden ham veri DB'ye yazılmıyor?

  • 32.000 INSERT/paket → I/O verimsizliği
  • Gereksiz DB boyutu büyümesi
  • Çözüm: Soğuk depolama (parquet disk) + Sıcak depolama (9 metrik DB)
📁

Neden pending/sending/sent/failed?

  • Program çöküp yeniden başlasa bile veri kaybolmaz
  • sending/: yarım kalan transfer recovery
  • In-memory queue değil: kalıcı durum
🔐

Neden SHA256 + UUID5?

  • SHA256: Veri bütünlüğü + idempotency anahtarı
  • UUID5: Deterministik ID → aynı veri = aynı UUID
  • Çift gönderimde duplicate oluşmaz
Gereksinimler

5 Gerçek Hayat Şartı

Görevde minimum 3 şart istenmiş. Tümü 5 şart uygulanmıştır.

  • ✓
    Idempotency — SHA256 checksum + UNIQUE constraint. Aynı paket iki kez gelirse {"ok":true,"duplicate":true} döner, yeni kayıt oluşmaz.
  • ✓
    Backpressure — pending/ klasöründe 2'den fazla paket birikirse yeni örnekleme durur, flush denenip 5 sn beklenir. Event: backpressure_triggered.
  • ✓
    Data Optimization — Parquet+Snappy+float32 ile %80 boyut azaltımı. DB'ye 9 metrik yazılır, 32.000 satır yazılmaz.
  • ✓
    Observability — Her iki serviste JSON structured logging. GET /metrics anlık sayaçlar. GET /health servis durumu. Log rotasyonu (7 gün).
  • ✓
    Data Corruption Check — Collector SHA256'yı diske yazıldıktan sonra hesaplar. Hub gelen bytes'tan SHA256 hesaplar. Bozuk parquet status='failed' olarak işaretlenir.

Test

12 Saatlik Simülasyon Testi

Hesap

12 saat = 43.200 saniye
Paket süresi = 10 saniye
Toplam paket = 4.320

acceleration=100:
  Örnekleme süresi  = 10s / 100 = 0.1s
  Toplam örnekleme  = 4320 × 0.1s = 432s ≈ 7 dk
  Gerçek süre       = ~10–20 dakika

Çalıştırma

# Varsayılan (hızlandırılmış, 100x)
docker compose up --build

# Gerçek zamanlı
docker compose run --rm collector \
  python -m app.collector \
  --mode realtime --hours 12

Test Kontrolleri

  • ✅ total = 4320, processed = 4320
  • ✅ Her başarılı paket: 3 event (received, validated, processed)
  • ✅ Parquet ~250–300 KB/paket (ham ~1.5 MB)
  • ✅ Orphaned event yok (FK bütünlüğü)
  • ✅ 9 metrik/paket DB'de

DB Doğrulama

-- Paket özeti
SELECT status, COUNT(*)
FROM packets
GROUP BY status;

-- FK bütünlüğü (0 dönmeli)
SELECT COUNT(*) orphaned
FROM packet_events pe
LEFT JOIN packets p
  ON pe.packet_id = p.packet_id
WHERE p.packet_id IS NULL;

5.1 Idempotency Testi

Ham veri DB'de yok, sadece metrikler var.

SELECT p.packet_id, m.mean_x,
       m.rms_x, m.peak_x
FROM packets p
JOIN packet_metrics m
  ON p.packet_id = m.packet_id
LIMIT 3;

Otomatik Temizleme Politikaları

Disk ve veritabanı dolmaması için üç ayrı katmanda otomatik temizleme mekanizması çalışır.

📂

sent/ Klasörü

50
MAX_SENT_PACKETS
  • Başarıyla gönderilen paketler tutulur
  • Her başarılı gönderi sonrası kontrol edilir
  • 50'yi aşınca en eski paketler silinir (FIFO)
  • .parquet + .json meta dosyası birlikte silinir
event: "packet_cleanup"
directory: "sent"
❌

failed/ Klasörü

100
MAX_FAILED_PACKETS
  • 5 deneme aşılan paketler buraya taşınır
  • Her failed taşıması sonrası kontrol edilir
  • 100'ü aşınca en eski paketler silinir (FIFO)
  • .parquet + .json meta dosyası birlikte silinir
event: "packet_cleanup"
directory: "failed"
🗄️

packet_events Tablosu

30
GÜN — RETENTION_DAYS

  • Her 24 saatte bir çalışır
  • 30 günden eski tüm event kayıtları silinir
  • packets ve packet_metrics korunur

Collector Cleanup Mantığı

# cleanup.py — FIFO silme
packet_files = sorted(dir.glob("*.parquet"),
    key=lambda p: p.stat().st_mtime)

# En eskiler başta — onlar silinir
to_delete = packet_files[: len - max_files]
for f in to_delete:
    f.unlink()            # .parquet
    meta_file.unlink()    # .json
